# OnboardingFlow Demo Script

## Opening

This is a separate proof-of-concept using sample data. It is not connected to OSHE systems.

I built it to make the onboarding workflow easier to review visually and to show one possible direction for tracking invites, applicant submissions, review steps, required documents, missing-information follow-up, and audit history.

This is not production-ready yet, but it demonstrates the workflow direction and what would need to be hardened for a real pilot.

## Demo Flow

1. Open the dashboard.
2. Show the invite list.
3. Open an onboarding invite.
4. Show the public onboarding form.
5. Show document requirements.
6. Upload a document and show it in SharePoint.
7. Show reviewer checklist and document review.
8. Mark an invite as Needs Info.
9. Send or preview the needs-info follow-up email.
10. Open the same public link in resubmission mode.
11. Resubmit updated information.
12. Show the activity log and reports.

## Short Spoken Lines

This is the dashboard.

This is an onboarding invite.

This is the public form the applicant sees.

The required document upload is stored in SharePoint.

The reviewer can mark documents as reviewed.

If information is missing, the reviewer can request an update.

The applicant can reopen the same link and resubmit.

The activity log shows the review trail.

## Closing

That is the main workflow. I kept this as a separate sample-data proof so it can be reviewed safely without touching any real OSHE systems.
